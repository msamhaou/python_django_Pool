  1. request:

    2. The user's browser sends an HTTP request to the server hosting the website, specifying the URL of the requested page.

  2. URL Resolution:

    3. Django's url dispatcher match the incoming url from the Http request with the proper view function

    the Url dispacher figure it out wich view to use trough the urls.py file

  3. View Processing :

    4. after the url resolved to a proper view function , Django apply the function

  4. Template Rendering :

    5. if the function is a rendering function , template contains HTML markup along with tags that generate content dynamically

    Django's template engine processes the template, replacing template tags and variables with their corresponding values.

  5. HTTP Response :

    6. once template is rendered, Django construct an Http Response contains the Html rendered page

    the response include a header indicate the content type (usually 'text/html')

    django sends Http response to the client

  6. Client Display :

    7. Finally, the client (e.g., web browser) receives the HTTP response and renders the HTML content.
